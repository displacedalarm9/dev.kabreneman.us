Section 10 — Maintenance Rules
─────────────────────────────────────────────────────────────────────────────────────────

  1. APPEND-ONLY: Entries are never removed. Retired artifacts change status to ARCHIVED.
  2. NEW AUID: Any new artifact must have its AUID added to Section 3 before or at publication.
  3. NEXT AUID: The next globally available AUID values, using the current class-scoped
     convention (AUID-<CLASS>-######). Note: this convention is flagged in R-00001_ as
     COL-003 (MEDIUM) for review — operator has not yet adopted a global sequential format.
     Until COL-003 is resolved, continue using class-scoped values:
       AUID-L-00002  (next L-class)
       AUID-P-00002  (next P-class)
       AUID-R-00002  (next R-class)
       AUID-REP-00002 (next REP-class)
       AUID-M-00001  (first M-class, when M-00001 is retroactively registered)
       AUID-X-00001  (reserved for ghost X-00001_ if created; do not use until then)
       AUID-X-00002  (proposed for X-00002_ retroactive registration, see R-00001_ COL-002)
       AUID-G-00002  (proposed for G-00002 retroactive registration, see R-00001_ COL-002)
  4. CURRENCY: Update the currency date at the top of this document on each revision.
  5. COLLISION CHECK: Before assigning any new AUID, consult Section 3 and Section 5
